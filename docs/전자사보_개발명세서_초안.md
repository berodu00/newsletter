# 고려아연 전자사보 고도화 프로젝트 - 최종 개발 명세서

## 📋 문서 정보
- **프로젝트명**: 고려아연(Korea Zinc) 전자사보 고도화 시스템
- **작성일**: 2025-12-25
- **개발 방식**: Vibe Coding (Gemini 3 Pro + Google AntiGravity)
- **대상 사용자**: 고려아연 및 계열사 임직원

---

## 1. 프로젝트 개요

### 1.1 추진 배경
- **노후 시스템 개선**: 오래된 시스템으로 최신 브라우저 환경 부적합, PDF 중심의 제한적 열람
- **모바일 접근성 부재**: 내부망 전용으로 외부 네트워크(출퇴근길 등)에서 접근 불가
- **소셜 컨텐츠 증가**: 고려아연 YouTube, Instagram 등 SNS 컨텐츠 표시 시스템 부재
- **분석 체계 미비**: 사보 소비율 감소 및 방문자/조회수 분석 도구 부재

### 1.2 프로젝트 목표
웹 기반 반응형 플랫폼으로 전환하여 **사보 열람 + SNS 컨텐츠 + 임직원 참여**를 통합한 **커뮤니티 플랫폼** 구축

### 1.3 프로젝트 범위
**이번 프로젝트는 "전자 사보 시스템"만 구축합니다.**
- 동호회 관리 시스템
- 특허 관리 시스템
- 공지 관리 시스템
→ 위 3개 시스템은 향후 확장 계획으로 별도 추진 예정

## 3. 시스템 아키텍처

### 3.1 사이트맵 구조

```
전자사보 시스템
│
├── 메인 페이지 (/)
│   ├── 롤링 배너 (최근 게시물)
│   ├── 팝업 (공지사항, 이벤트 알림)
│   └── 카테고리별 최근 게시물 Carousel
│
├── 웹진 (/magazine)
│   ├── 전체 목록
│   ├── 카테고리별 조회 (Special, People, Life)
│   ├── 해시태그별 조회
│   ├── 검색 기능
│   └── 컨텐츠 상세 페이지
│       ├── 조회수 표시
│       ├── 별점 (1~5점)
│       ├── 반응 이모지 (좋아요, 슬퍼요, 화나요, 웃겨요)
│       └── 댓글 (선택적 기능, 개발 중 판단)
│
├── 소셜 컨텐츠 (/social)
│   ├── 유튜브 게시판 (YouTube Data API 연동)
│   ├── 인스타그램 게시판 (Instagram Graph API 연동)
│   └── iframe 임베드 (고려아연 공식 홈페이지에 삽입 가능)
│
├── 이벤트 (/events)
│   ├── 진행중 이벤트 목록
│   ├── 종료된 이벤트 목록
│   ├── 이벤트 상세 및 참여 페이지
│   └── 당첨자 발표 페이지
│
├── 아이디어 제안 (/ideas)
│   └── 제안 폼 (커뮤니케이션팀으로 전달)
│
└── 관리자 페이지 (/admin)
    ├── 대시보드
    ├── 컨텐츠 관리 (CRUD)
    ├── 이벤트 관리
    ├── 팝업 관리
    ├── 카테고리/해시태그 마스터 관리
    └── 사용자 통계 조회
```

### 3.2 데이터베이스 스키마 (주요 테이블)

#### 3.2.1 users (사용자)
```sql
CREATE TABLE users (
    user_id SERIAL PRIMARY KEY,
    username VARCHAR(100) NOT NULL UNIQUE,  -- SSO 연동 ID
    name VARCHAR(100),
    email VARCHAR(255),
    department VARCHAR(100),
    role VARCHAR(20) DEFAULT 'USER',  -- USER, ADMIN
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login TIMESTAMP
);
```

#### 3.2.2 categories (카테고리 마스터)
```sql
CREATE TABLE categories (
    category_id SERIAL PRIMARY KEY,
    category_name VARCHAR(50) NOT NULL UNIQUE,  -- Special, People, Life
    display_order INT DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.2.3 hashtags (해시태그 마스터)
```sql
CREATE TABLE hashtags (
    hashtag_id SERIAL PRIMARY KEY,
    hashtag_name VARCHAR(50) NOT NULL UNIQUE,
    usage_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.2.4 contents (사보 컨텐츠)
```sql
CREATE TABLE contents (
    content_id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    summary TEXT,  -- 2줄 미만 요약
    body_html TEXT NOT NULL,  -- HTML 본문
    thumbnail_url VARCHAR(500),  -- 썸네일 이미지 경로
    category_id INT REFERENCES categories(category_id),
    author_id INT REFERENCES users(user_id),
    view_count INT DEFAULT 0,
    average_rating DECIMAL(2,1) DEFAULT 0.0,  -- 평균 별점 (0.0 ~ 5.0)
    is_published BOOLEAN DEFAULT FALSE,
    published_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.2.5 content_hashtags (컨텐츠-해시태그 연결)
```sql
CREATE TABLE content_hashtags (
    content_id INT REFERENCES contents(content_id) ON DELETE CASCADE,
    hashtag_id INT REFERENCES hashtags(hashtag_id) ON DELETE CASCADE,
    PRIMARY KEY (content_id, hashtag_id)
);
```

#### 3.2.6 reactions (반응)
```sql
CREATE TABLE reactions (
    reaction_id SERIAL PRIMARY KEY,
    content_id INT REFERENCES contents(content_id) ON DELETE CASCADE,
    user_id INT REFERENCES users(user_id),
    reaction_type VARCHAR(20) NOT NULL,  -- LIKE, SAD, ANGRY, FUNNY
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(content_id, user_id, reaction_type)
);
```

#### 3.2.7 ratings (별점)
```sql
CREATE TABLE ratings (
    rating_id SERIAL PRIMARY KEY,
    content_id INT REFERENCES contents(content_id) ON DELETE CASCADE,
    user_id INT REFERENCES users(user_id),
    rating_value INT CHECK (rating_value BETWEEN 1 AND 5),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(content_id, user_id)
);
```

#### 3.2.8 social_contents (SNS 연동 컨텐츠)
```sql
CREATE TABLE social_contents (
    social_content_id SERIAL PRIMARY KEY,
    platform VARCHAR(20) NOT NULL,  -- YOUTUBE, INSTAGRAM
    external_id VARCHAR(255) NOT NULL UNIQUE,  -- YouTube Video ID, Instagram Post ID
    title VARCHAR(255),
    description TEXT,
    thumbnail_url VARCHAR(500),
    media_url VARCHAR(500),
    published_at TIMESTAMP,
    fetched_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    view_count INT DEFAULT 0
);
```

#### 3.2.9 events (이벤트)
```sql
CREATE TABLE events (
    event_id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    thumbnail_url VARCHAR(500),
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_by INT REFERENCES users(user_id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.2.10 event_participants (이벤트 참여자)
```sql
CREATE TABLE event_participants (
    participant_id SERIAL PRIMARY KEY,
    event_id INT REFERENCES events(event_id) ON DELETE CASCADE,
    user_id INT REFERENCES users(user_id),
    comment TEXT,
    is_winner BOOLEAN DEFAULT FALSE,
    participated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(event_id, user_id)
);
```

#### 3.2.11 popups (팝업 관리)
```sql
CREATE TABLE popups (
    popup_id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    content TEXT,
    image_url VARCHAR(500),
    link_url VARCHAR(500),
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    display_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.2.12 ideas (아이디어 제안)
```sql
CREATE TABLE ideas (
    idea_id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(user_id),
    title VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    status VARCHAR(20) DEFAULT 'PENDING',  -- PENDING, REVIEWED, ACCEPTED, REJECTED
    admin_comment TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    reviewed_at TIMESTAMP
);
```

#### 3.2.13 visitor_logs (방문 로그)
```sql
CREATE TABLE visitor_logs (
    log_id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(user_id),
    visited_at DATE NOT NULL,
    page_views INT DEFAULT 1,
    UNIQUE(user_id, visited_at)
);
```

#### 3.2.14 resource_files (업로드 리소스)
```sql
CREATE TABLE resource_files (
    file_id SERIAL PRIMARY KEY,
    file_name VARCHAR(255) NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    file_size BIGINT,
    mime_type VARCHAR(100),
    uploaded_by INT REFERENCES users(user_id),
    uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 4. 주요 기능 명세

### 4.1 사용자 기능 (Frontend)

#### 4.1.1 인증 및 접근
- **전용 로그인 페이지**
  - SSO(Single Sign-On) 연동 (프로토콜은 개발 중 선정: SAML 2.0 또는 OAuth 2.0)
  - AD 계정 없는 생산직 및 계열사 임직원을 위한 대체 인증 방식 지원
- **외부 네트워크 접근 지원**
  - HTTPS 적용
  - 출퇴근길 등 모바일 환경에서 접속 가능

#### 4.1.2 메인 페이지
- **롤링 배너**: 최근 게시물 자동 슬라이드
- **팝업 관리**: 신규 컨텐츠, 이벤트 알림 팝업 표시
- **카테고리별 Carousel**: Special, People, Life 각 카테고리 최신 게시물 3~5개씩 표시

#### 4.1.3 웹진 (사보 컨텐츠)
- **목록 조회**
  - 카테고리별 필터링 (Special, People, Life)
  - 해시태그별 필터링 (클릭 시 해당 태그 포함 게시물 조회)
  - 검색 기능 (제목, 본문 검색)
  - 페이징 처리 (페이지당 12개 또는 20개, 개발 중 결정)
- **상세 페이지**
  - HTML 본문 렌더링
  - 조회수 자동 증가
  - 별점 부여 (1~5점, 사용자당 1회)
  - 반응 이모지 (좋아요, 슬퍼요, 화나요, 웃겨요 - 중복 선택 가능 여부는 개발 중 결정)
  - 관련 컨텐츠 추천 (동일 카테고리 또는 해시태그)

#### 4.1.4 소셜 컨텐츠
- **유튜브 게시판**
  - YouTube Data API 연동으로 고려아연 채널 최신 영상 자동 수집
  - 썸네일, 제목, 재생 링크 표시
  - 클릭 시 YouTube 플레이어 모달 또는 새 창
- **인스타그램 게시판**
  - Instagram Graph API 연동으로 고려아연 계정 최신 포스트 자동 수집
  - 이미지, 캡션 표시
  - 클릭 시 Instagram 링크로 이동
- **iframe 임베드**
  - 고려아연 공식 홈페이지에 소셜 컨텐츠 게시판을 iframe으로 삽입 가능
  - iframe 전용 경량 페이지 제공 (/social/embed)

#### 4.1.5 이벤트
- **진행중 이벤트 목록**: 현재 날짜 기준 진행 중인 이벤트만 표시
- **종료된 이벤트 목록**: 과거 이벤트 아카이브
- **이벤트 참여**
  - 참여 버튼 클릭 (로그인 사용자만)
  - 선택적 코멘트 입력
  - 중복 참여 방지
- **당첨자 발표**
  - 관리자가 랜덤 추첨한 당첨자 목록 공개
  - 당첨 여부 개인별 확인 기능

#### 4.1.6 아이디어 제안
- **제안 폼**
  - 제목, 설명 입력
  - 커뮤니케이션팀 관리자 페이지로 자동 전달
  - 제안 내역 조회 (본인이 작성한 제안만)
  - 검토 상태 표시 (대기중, 검토완료, 승인, 거부)

#### 4.1.7 반응형 디자인
- **PC 우선**: 1920x1080 기준 최적화
- **모바일 대응**: 768px 이하에서 레이아웃 자동 조정
- **터치 UI**: 모바일 환경에서 버튼 크기, 간격 최적화

### 4.2 관리자 기능 (Backend)

#### 4.2.1 컨텐츠 관리
- **게시물 등록**
  - 제목, 카테고리 선택, 해시태그 입력 (쉼표 구분)
  - HTML 에디터 (CKEditor 5)로 본문 작성
  - 썸네일 이미지 업로드
  - 요약 정보 입력 (2줄 미만 텍스트)
  - 발행 상태 선택 (임시저장 / 즉시발행)
- **게시물 수정 / 삭제**
  - 기존 게시물 조회 및 편집
  - 조회수, 별점 통계 유지
- **리소스 업로드**
  - 이미지 파일 업로드 (JPG, PNG, GIF, WebP)
  - 업로드 후 파일 경로 자동 복사 (에디터에 붙여넣기 용)
  - 파일 목록 관리 (삭제 기능 포함)

#### 4.2.2 팝업 관리
- **팝업 등록**
  - 제목, 내용, 이미지, 링크 URL
  - 노출 기간 설정 (시작일, 종료일)
  - 노출 순서 지정
- **활성화 / 비활성화**: 즉시 적용

#### 4.2.3 이벤트 관리
- **이벤트 등록**
  - 제목, 설명, 썸네일, 이벤트 기간
  - 활성화 상태 설정
- **참여 현황 조회**
  - 참여자 목록 테이블 형식 표시
  - 참여자 목록 CSV 다운로드
- **당첨자 랜덤 추첨**
  - 추첨 인원 수 입력 (예: 10명)
  - "추첨하기" 버튼 클릭 시 랜덤 선정
  - 당첨자 is_winner 플래그 업데이트
- **당첨자 발표**
  - 당첨자 목록 공개 처리
  - 사용자 페이지에 당첨자 이름 표시

#### 4.2.4 카테고리 / 해시태그 마스터 관리
- **카테고리 관리**
  - 기본 카테고리: Special, People, Life (수정 불가, 삭제 불가)
  - 추가 카테고리 등록 가능 (선택 사항)
  - 노출 순서 조정
- **해시태그 관리**
  - 자동 생성: 게시물 작성 시 입력된 해시태그 자동 등록
  - 사용 횟수 집계
  - 수동 삭제 가능 (게시물 연결 해제)

#### 4.2.5 대시보드
- **조회수 TOP 10**: 게시물 제목, 조회수, 링크
- **별점 TOP 10**: 게시물 제목, 평균 별점, 링크
- **방문자 수 추이**
  - 일별 방문자 수 그래프 (최근 30일)
  - 주별 집계 (선택 가능)
  - 월별 집계 (선택 가능)
- **카테고리별 선호도**
  - Special, People, Life 각 카테고리별 조회수 비율 (파이 차트)
- **해시태그별 선호도**
  - 상위 20개 해시태그 사용 횟수 (막대 그래프 또는 워드클라우드)
- **반응 통계**
  - 좋아요, 슬퍼요, 화나요, 웃겨요 각각 총 횟수 (도넛 차트)
- **Naver Analytics 연동**
  - Analytics 대시보드 링크 제공
  - 추가 상세 분석은 Analytics에서 확인

#### 4.2.6 아이디어 제안 관리
- **제안 목록 조회**
  - 상태별 필터 (대기중, 검토완료, 승인, 거부)
  - 작성자, 작성일 표시
- **제안 검토**
  - 제안 내용 확인
  - 상태 변경 (검토완료, 승인, 거부)
  - 관리자 코멘트 입력
  - 제안자에게 알림 (선택 사항, 개발 중 결정)

## 5. 비기능적 요구사항

### 5.1 성능
- **페이지 로딩 시간**: 3초 이내 (네트워크 조건: 50Mbps 기준)
- **동시 접속자**: 최소 500명 지원
- **이미지 최적화**
  - 썸네일 자동 생성 (200x200px)
  - WebP 포맷 권장 (PNG/JPG도 허용)
  - 용량 제한: 5MB/파일

### 5.2 보안
- **HTTPS**: SSL 인증서 적용
- **SSO 연동**: 인증 프로토콜은 개발 중 선정 (SAML 2.0 또는 OAuth 2.0 중 적합한 방식)
- **SQL Injection 방지**: Prepared Statement 사용 (Spring Data JPA 자동 처리)
- **XSS 방지**: HTML 에디터 입력 값 Sanitization
- **CSRF 방지**: Spring Security CSRF Token 활용
- **권한 관리**: USER / ADMIN 역할 기반 접근 제어

### 5.3 접근성
- **크로스 브라우징**: Chrome, Edge, Safari, Firefox 최신 버전 지원
- **모바일 터치 UI**: 버튼 최소 크기 44x44px (애플 HIG 권장)
- **키보드 네비게이션**: Tab 키로 모든 주요 기능 접근 가능
- **대체 텍스트**: 이미지에 alt 속성 필수

### 5.4 유지보수성
- **코드 컨벤션**: Google Java Style Guide 준수
- **주석**: 복잡한 로직에 한글 주석 추가
- **Git Commit**: 기능 단위로 커밋, AI 태그 활용 (예: `[AI-FEATURE] 사보 목록 조회 API 구현`)
- **로깅**: SLF4J + Logback 사용, ERROR/WARN/INFO 레벨 구분

## 6. API 명세 (주요 엔드포인트)

### 6.1 인증
- `POST /api/auth/login`: SSO 로그인
- `POST /api/auth/logout`: 로그아웃
- `GET /api/auth/me`: 현재 사용자 정보 조회

### 6.2 컨텐츠 (사보)
- `GET /api/contents`: 목록 조회 (카테고리, 해시태그, 페이징 파라미터)
- `GET /api/contents/{id}`: 상세 조회 (조회수 자동 증가)
- `POST /api/contents`: 등록 (관리자 전용)
- `PUT /api/contents/{id}`: 수정 (관리자 전용)
- `DELETE /api/contents/{id}`: 삭제 (관리자 전용)
- `POST /api/contents/{id}/rating`: 별점 부여
- `POST /api/contents/{id}/reaction`: 반응 이모지 등록

### 6.3 소셜 컨텐츠
- `GET /api/social/youtube`: 유튜브 컨텐츠 목록
- `GET /api/social/instagram`: 인스타그램 컨텐츠 목록
- `POST /api/social/sync`: SNS 컨텐츠 동기화 (관리자 전용, 수동 실행)

### 6.4 이벤트
- `GET /api/events`: 이벤트 목록 (진행중 / 종료 필터)
- `GET /api/events/{id}`: 상세 조회
- `POST /api/events/{id}/participate`: 참여 신청
- `POST /api/events/{id}/draw`: 당첨자 추첨 (관리자 전용)
- `GET /api/events/{id}/winners`: 당첨자 목록 조회

### 6.5 팝업
- `GET /api/popups`: 활성 팝업 목록 (현재 날짜 기준)
- `POST /api/popups`: 등록 (관리자 전용)
- `PUT /api/popups/{id}`: 수정 (관리자 전용)
- `DELETE /api/popups/{id}`: 삭제 (관리자 전용)

### 6.6 아이디어 제안
- `GET /api/ideas`: 목록 조회 (본인 작성분만 또는 관리자는 전체)
- `POST /api/ideas`: 제안 등록
- `PUT /api/ideas/{id}/review`: 검토 처리 (관리자 전용)

### 6.7 대시보드
- `GET /api/dashboard/top-views`: 조회수 TOP 10
- `GET /api/dashboard/top-ratings`: 별점 TOP 10
- `GET /api/dashboard/visitor-trend`: 방문자 추이 (일/주/월 파라미터)
- `GET /api/dashboard/category-stats`: 카테고리별 통계
- `GET /api/dashboard/hashtag-stats`: 해시태그별 통계
- `GET /api/dashboard/reaction-stats`: 반응 통계

## 8. 추가 고려사항

### 8.1 향후 확장 가능성
- **동호회 관리 시스템**: 사보 시스템과 별도 모듈로 추가 가능
- **특허 관리 시스템**: 독립적인 시스템으로 구축 가능
- **공지 관리 시스템**: 팝업 기능 확장 또는 별도 게시판 추가

### 8.2 운영상 주의사항
- **SNS API 쿼터**: YouTube Data API는 일일 10,000 units, Instagram Graph API는 200 requests/hour 제한
- **배치 스케줄링**: SNS 컨텐츠 동기화는 1일 1회 (새벽 2시 권장)
- **백업**: PostgreSQL 자동 백업 설정 (일일 1회)

### 8.3 개발 중 판단이 필요한 사항 (Gemini가 질문할 항목)
1. **댓글 기능**: 사보 컨텐츠에 댓글을 달 수 있도록 할지?
   - Yes: comments 테이블 추가 필요
   - No: 반응 이모지와 별점만으로 충분
2. **반응 이모지 중복 선택**: 사용자가 여러 이모지를 동시에 누를 수 있을지?
   - Yes: 현재 스키마 유지
   - No: UNIQUE 제약조건 수정 필요
3. **SNS 컨텐츠 자동 동기화 주기**: 1일 1회 vs 실시간(1시간마다)?
4. **대시보드 차트 스타일**: 애니메이션 효과 포함 여부?
5. **아이디어 제안 알림**: 이메일 알림 vs 관리자 페이지 내 알림만?
